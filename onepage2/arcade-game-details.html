Classic Arcade Game Details


Project Overview

The goal of this Udacity’s project is to develop a simple arcade game using JavaScript object-oriented programming features. Visual assets and a game loop engine code were provided for the beginning.

Implementation Details

Implementation


We need to add some functionality to the starter code and develop next entities and conditions:

  • Develop the “Hero".
  • Develop the "Enemies".
  • Develop the collisions and win conditions.
  • Describe the game reset logic.

Developing the "Hero"

In file app.js: `class Hero ` consist of a constructor function and methods:

  • `constructor()` defines the hero's initial position on the board, an image we use to render the hero, and a boolean operator which will be used later as a part of the resetting process.
  • `update()` method contains:
    • The collision condition between the hero and enemies.
    • The win condition if the hero reaches the water.
    • Resets hero position to an initial if any of them has occurred
`reset()` function contains the hero's initial coordinates on the board. `render()` method renders the hero at an initial position.
    `handleInput()` function:
  • Updates the hero's x/y coordinates accordingly to pressed key and width/height of game board blocks.
  • `if` conditions in lines 67, 72, 77, and 82 keep the hero inside the game board
`const player = new Hero();` in line 92, instantiates the Hero object;


Developing the "Enemies"

    In `var Enemy = function(x, y, speed)`:
  • `this.x = x; this.y = y;` - set a placeholder for an enemy's initial position.
  • `this.speed` - set an enemy's speed
Then in lines 95 - 101, we set up six enemy objects and create an `allEnemies` array.
    `Enemy.prototype.update = function(dt)`:
  • Contains a condition that allows the enemies to move away from the board.
  • After that, returns enemies to the start.
  • Randomizes initial enemies' position by x/y axis.
`Enemy.prototype.render = function()` - draws the enemies on the screen.


Developing Collisions and Win Conditions

In lines 42 - 43, we loop through `allEnemies` array and compare their current y-coordinates with the hero's current position. If they match, we consider it as a collision and reset the hero's position to initial in line 44 by `this.reset()`.

The user wins when the hero reaches the water. So the winning condition is `this.y < 20`. We check it in lines 46-52. Then, in the hero's constructor function, we assign `true` to `this.you`, and return the hero to the start position.


Game Reset

In engine.js, in lines 11 and 12, we create two variables: `modal` and `playAgain`. We use them for the modal window.

In line 45, we check the boolean `youWin`. If it evaluates to `true`:

  • We stop game animation with `win.cancelAnimationFrame(gameOver);`
  • Add class `visible` to the `div' in line 11 in index.html and display a pop-up window with the `Play Again` button.
If `youWin` evaluates to `false`, the game continues with `gameOver = win.requestAnimationFrame(main)` in lines 48-50.

Function `playAgain.addEventListener` in line 14 adds an event listener to the `Play Again` button, and after pushing it:

  • Toggles off the modal window.
  • Changes `youWin` to `false`.
  • Restarts the animation loop.
